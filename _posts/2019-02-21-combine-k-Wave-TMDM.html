--- title: Implementation of k-Wave for arbitrary shape transducer ---


This example shows how to implement the k-Wave for simulations with curved transducer. Currently, only phased array transducer can be directly simulated in the mSOUND to generate focused beams. If other transducer settings are required, people can first use k-Wave to propagate the wave field for a small distance and record the data at a plane. Then implement the recorded data at that plane to the mSOUND as source pressure. In this example, a curved transduce is first modeled with k-Wave and then continues with the mSOUND.

Generate the source pressure with k-Wave


Define the computational domain in the k-Wave calculation.
dx = 1.8750e-04; % step size in the x direction [m]
dy = 1.8750e-04; % step size in the y direction [m]
dt = 3.1250e-08; % time step size [s]
Nx = 240; % total computatoinal domain size in x direction [grid points]
Ny = 40; % total computatoinal domain size in y direction [grid points]
kgrid = kWaveGrid(Nx, dx, Ny, dy);

Define a curved transducer with k-Wave

In this example, a curved transducer is defined with k-Wave using the function makeCircle.

TR_radius = 0.0060; % Transducer radius [m]
TR_focus = 0.0150; % Transducer focal length [m]
focus = round(TR_focus/dx); % Transducer focal length [grid points]
cx = round(Nx/2); % x coordinate of the tranducer center[grid points]
cy = 10 + focus; % y coordinate of the tranducer center[grid points]
arc_angle1 = asin(TR_radius/TR_focus); % [radians]
circle1 = makeCircle(Nx, Ny, cx, cy, focus, arc_angle1, false);
arc_angle2 = pi*2; % [radians]
circle2 = makeCircle(Nx, Ny, cx, cy, focus, arc_angle2, false);
arc_angle3 = pi*2 - arc_angle1; % [radians]
circle3 = makeCircle(Nx, Ny, cx, cy, focus, arc_angle3, false);
circle = circle2 - circle3 + circle1;

source.p_mask = zeros(Nx, Ny);
source.p_mask = circle; % define source mask
sensor.mask = zeros(Nx, Ny);
sensor.mask(:, Ny/2+offcenter) = 1; % define sensor mask

Excitation signal

A four-cycles Gaussian-modulated pulse is used as the excitation signal.

source_mag = 1; % [Pa]
fc = 1.0e6; % [Hz]
ts = [-num_cycles/fc:dt:num_cycles/fc]; % excitation pulse length
excit_p = sin(2*pi*fc*(ts)).*exp(-(ts).^2*fc^2/2); % excitation
source.p = excit_p*source_mag;
input_args = {'DisplayMask', source.p_mask, 'DataCast', 'single'};
% run the simulation
sensor_data = kspaceFirstOrder2D(kgrid, medium, source, sensor, input_args{:});
% Reshape the output data for further simulation
sensor_data = reshape(sensor_data, Nx, length(kgrid.t_array));

Implement signals to the TMDM

Pressure recorded at the sensor positions from k-Wave simulation will be used as the source pressure for the TMDM simulation. One thing to be notice is that sensor data from k-Wave is in indexed as [time, sensor_positions], while in the TMDM is [sensor_positions, time]. If the time step or transverse saptial domain size in the k-Wave is different from TMDM, sensor_data from the k-Wave needs to be resized.

xlength = kgrid.dx*kgrid.Nx;
ylength = TR_focus - kgrid.dy*(Ny-20);
tlength = max(kgrid.t_array);
dt = 6.2500e-08; % a larger time step is applied
mgrid = set_grid(dt, tlength, dx, xlength, dy, ylength);
% need to resize the sensor data as a large time step is appled in the TMDM
source_p = sensor_data(:, 1:2:end).';

% medium properties in the TMDM
medium.c = 1500; % speed of sound [m/s]
medium.rho = 1000; % density [kg/m^3]
medium.beta = 0.0; % nonlinear coefficient
medium.ca = 0; % attenuation coefficient [dB/(MHz^y cm)]
medium.cb = 2.0; % power law exponent
medium.c0 = 1500; % reference speed of sound [m/s]

% sensor mask in the TMDM
sensor_mask = zeros(mgrid.num_x, mgrid.num_y);
sensor_mask(:, end) = 1;

Results

Now we finish the simulation with TMDM. The figure below shows the time domain results at the geometrical focus of the transducer and the result generated with TMDM is compared with the result directly simulated with k-Wave.
% run the simulation with TMDM
p_time = Forward2D(mgrid, medium, source_p, sensor_mask);
% Reshape the data from simulation
p_time = reshape(p_time, mgrid.num_t, mgrid.num_x);
